# design-brief 스킬 스펙

## 목적

`design-brief`는 vague product 또는 interface idea를 design-ready brief로 정리하는 스킬입니다.
핵심 산출물은 user outcome, hierarchy, tone, constraints, non-goals, directional decisions가 잠긴 brief입니다.

## 경계

- 포함:
  - 목표 사용자와 desired outcome 정의
  - hierarchy, tone, constraints, non-goal 정리
  - 방향성 결정을 위한 핵심 질문 고정
- 제외:
  - Figma execution
  - code implementation
  - existing UI critique

## 처리하려는 작업 형태

- vague idea를 design direction으로 바꿔야 하는 경우
- design work 전에 무엇을 만들고 무엇을 만들지 않을지 잠가야 하는 경우

## 엔트리포인트 / 대표 표면

- 대표 표면: `designer-kit/skills/design-brief/SKILL.md`
- 관련 상위 라우팅: `designer-kit-guide`

## 핵심 처리 계약

- brief는 outcome, audience, hierarchy, tone, constraints, non-goals를 포함해야 한다.
- implementation detail이 아니라 design decision 기준을 남겨야 한다.
- unresolved question이 있더라도 다음 design stage가 가능한 수준까지 방향을 잠근다.

## 독립성 원칙

- 이 스킬은 critique나 screen specification을 소유하지 않는다.
- brief artifact만 단독으로 읽어도 design direction이 이해 가능해야 한다.

## 확장 원칙

- 새 규칙은 design brief 품질을 높이는 경우에만 추가한다.
- screen-level 구조나 critique lens는 해당 sibling skill spec에 남긴다.
