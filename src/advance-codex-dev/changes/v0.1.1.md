# advance-codex-dev v0.1.1 Change Spec

## 변경사항 요약

- disallowed temporary-folder guidance를 제거하고 session artifacts의 기본 위치를 `.agents/sessions/{YYYYMMDD}/`로 정렬한다.
- `agents-sessions` skill을 추가해 `.agents/sessions` 폴더의 목적과 `turn-gate` plan/flow record 골격을 설명한다.
- `session-manager` skill, controller, tests, record schema, metadata를 제거하고 `turn-gate`를 active session flow의 소유 표면으로 둔다.
- `workflow-kit` autopilot state guidance도 같은 date-scoped session folder convention으로 맞춘다.
- `subagent-gate` skill을 추가해 subagent 호출 전 종료 시점 계획과 subagent 관점의 최소 context packet을 잠근다.
- empirical test 결과를 반영해 `subagent-gate` handoff packet의 main-thread blocked state와 no-handoff routing note를 명시한다.
- 2차 empirical test 결과를 반영해 no-handoff routing note의 필드 모양을 reason, route, next action, blocked state, re-entry condition으로 고정한다.

## 사용자 스펙 의도

- 이 저장소의 플러그인들이 더 이상 disallowed temporary folder를 사용하거나 권장하지 않게 하고 싶다.
- `.agents/sessions` 폴더의 기본 규격을 `turn-gate`의 날짜 기반 `000-plan.md`와 `001-*` flow record 골격에 맞추고 싶다.
- 별도 session record schema와 controller를 제공하는 `session-manager`는 기본 session flow와 중복되므로 제거하고 싶다.
- `advance-codex`의 session folder guidance는 `turn-gate`를 대체하지 않고, 폴더 의미와 경계만 설명해야 한다.
- subagent를 준비할 때는 먼저 언제 돌아와야 하는지와 무엇을 넘겨야 하는지를 명확히 잠그고 싶다.

## 변경 목적

`v0.1.1`은 session artifact 기능을 새 controller 중심으로 확장하는 release가 아니라, 기존 guidance를 `turn-gate` 중심의 session folder convention으로 정리하는 patch입니다.
기본 골격은 `.agents/sessions/{YYYYMMDD}/000-plan.md`와 `.agents/sessions/{YYYYMMDD}/{count-pad3}-{eng-lower-slug}.md`입니다.
`advance-codex`는 이 폴더의 목적과 경계를 설명하고, active turn continuity와 flow record 운영은 `loop-kit:turn-gate`가 소유합니다.

## 변경 상세

### Temporary Folder Guidance 제거

- `workflow-kit-dev`와 release `workflow-kit`의 `autopilot` state guidance를 date-scoped `.agents/sessions/{YYYYMMDD}/` 경로로 바꾼다.
- `advance-codex-dev`와 release `advance-codex`에서 legacy scratch-folder guidance를 제거한다.
- 검증 기준은 runtime plugin surfaces에서 disallowed temporary-folder guidance가 남지 않는 것이다.

### `agents-sessions` 추가

- `agents-sessions` skill을 추가해 `.agents/sessions` 폴더의 기본 용도와 `turn-gate` 정렬 골격을 설명한다.
- `advance-codex-dev-guide`가 session folder convention 요청을 `agents-sessions`로 라우팅하게 한다.
- plugin spec, guide spec, README, manifest prompt를 session folder convention 중심으로 갱신한다.

### `session-manager` 제거

- `session-manager` skill, spec, controller, tests, agent metadata, record format reference를 제거한다.
- `session-manager`의 legacy scratch-folder guidance를 제거한다.

### Cross-Plugin Alignment

- `workflow-kit-dev`와 release `workflow-kit`의 `autopilot` skill/spec이 `.agents/sessions/{YYYYMMDD}/` convention을 사용하도록 맞춘다.
- 이 동반 변경은 `advance-codex`의 session folder convention이 `workflow-kit` guidance와 충돌하지 않게 하기 위한 범위다.

### `subagent-gate` 추가

- `subagent-gate` skill을 추가해 subagent 호출 전 handoff gate를 소유하게 한다.
- 종료 시점 계획, stop boundary, output contract, edit permission, close plan을 먼저 작성하도록 한다.
- main-thread blocked state를 handoff packet의 명시 필드로 둔다.
- `subagent-gate` 경계 밖의 요청은 handoff로 억지 변환하지 않고 no-handoff routing note로 돌려보낸다.
- no-handoff routing note에는 reason, right skill or workflow, main-thread next action, main-thread blocked state, re-entry condition을 둔다.
- no-handoff 상태에서는 아직 없는 subagent ownership이나 pass/fail criteria를 억지로 채우지 않는다.
- main agent의 전체 맥락이 아니라 subagent 시점에서 필요한 user goal, verified facts, assigned files, constraints, pass/fail criteria, forbidden actions만 context packet으로 정리하게 한다.
- `subagent-creator`는 custom agent definition을 계속 소유하고, `subagent-gate`는 runtime handoff 준비를 소유한다.

## 비목표

- `turn-gate` flow record 생성/갱신 규칙을 `agents-sessions`에 복제하거나 재정의하지 않는다.
- custom agent TOML 정의 규칙을 `subagent-gate`에 흡수하지 않는다.
- 별도 `session_record.md`, `change_record.md`, `retrospective_record.md` schema를 유지하지 않는다.
- `.agents/sessions`를 plugin source, release surface, reusable spec의 대체 위치로 만들지 않는다.
- `workflow-kit`의 autopilot loop 자체를 재설계하지 않는다.

## 호환성 및 마이그레이션

- `session-manager`를 사용하던 호출자는 더 이상 이 plugin에서 해당 skill이나 controller를 찾을 수 없다.
- active turn continuity와 flow 기록은 `loop-kit:turn-gate`를 사용한다.
- `.agents/sessions`의 폴더 의미만 확인해야 할 때는 `agents-sessions`를 사용한다.
- autopilot state/resume artifacts는 date-scoped `.agents/sessions/{YYYYMMDD}/` 아래에 둔다.

## 검증 기준

- runtime skill source와 release surface 모두에서 `session-manager` runtime surface가 없어야 한다.
- runtime skill source와 release surface 모두에서 disallowed temporary-folder guidance가 없어야 한다.
- `advance-codex` release surface에 `agents-sessions` skill과 agent metadata가 포함되어야 한다.
- `advance-codex` release surface에 `subagent-gate` skill이 포함되어야 한다.
- `workflow-kit` dev/release `autopilot` guidance가 `.agents/sessions/{YYYYMMDD}/`를 사용해야 한다.
- release surface에는 `specs/` 디렉터리가 포함되지 않아야 한다.

## 정식 규칙 승격 여부

- 정식 규칙으로 승격할 항목: change spec 경로와 요약-first 문서 형식은 repo-level `AGENTS.md`와 `docs/templates/change-spec.md`가 소유한다.
- plugin/skill contract로 승격할 항목: `agents-sessions`의 folder-boundary contract는 `src/advance-codex-dev/specs/skills/agents-sessions.md`가 소유한다.
- plugin/skill contract로 승격할 항목: `subagent-gate`의 handoff gate contract는 `src/advance-codex-dev/specs/skills/subagent-gate.md`가 소유한다.
- change spec에만 남길 이력: `session-manager` 제거 배경, temporary-folder guidance 제거 과정, `workflow-kit` autopilot 동반 정렬의 release history.
