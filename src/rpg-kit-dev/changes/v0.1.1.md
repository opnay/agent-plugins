# rpg-kit-dev v0.1.1 Change Spec

## 변경사항 요약

- `rpg-kit-dev` version을 `0.1.1`로 올린다.
- `subagent-role`을 직접 실행 표면으로 둔다.
- README, plugin spec, manifest prompt가 plugin-level 선택 기준을 소유하게 한다.

## 사용자 스펙 의도

- manifest prompt, README, plugin spec에서 plugin 사용 기준을 분명히 하고 싶다.
- `rpg-kit-dev`는 이번 `main` merge 이후 첫 수정이므로 patch version bump를 적용한다.

## 변경 목적

`rpg-kit-dev`의 실행 표면을 `subagent-role` 하나로 좁힙니다.
plugin-level 설명과 선택 기준은 `README.md`, `specs/plugin.md`, `.codex-plugin/plugin.json`에 둡니다.

## 변경 상세

### Plugin-Level Routing 정리

- `subagent-role` spec의 sibling 의존 문구를 직접 실행 표면 기준으로 갱신한다.
- evaluation spec의 plugin 사용 기준 참조를 plugin spec과 manifest 참조로 바꾼다.

### Version Bump

- 사용자 승인에 따라 `rpg-kit-dev` version을 `0.1.0`에서 `0.1.1`로 올린다.
- release `rpg-kit` surface도 build 산출물로 갱신한다.

## 비목표

- `subagent-role` skill의 role packet 계약을 바꾸지 않는다.
- subagent spawn 정책이나 runtime approval boundary를 완화하지 않는다.

## 호환성 및 마이그레이션

- 사용자는 `$rpg-kit:subagent-role`을 직접 호출한다.
- role packet, answer contract, learning note 기능은 `subagent-role`에 유지된다.

## 검증 기준

- dev/release surface는 `subagent-role`을 제공해야 한다.
- dev/release manifest version은 `0.1.1`이어야 한다.
- release surface에는 `specs/`와 `changes/`가 없어야 한다.

## 정식 규칙 승격 여부

- normative plugin contract는 `src/rpg-kit-dev/specs/plugin.md`와 `src/rpg-kit-dev/specs/skills/subagent-role.md`가 소유한다.
- change spec에만 남길 이력: patch bump 근거와 plugin 사용 기준 정리 범위.
